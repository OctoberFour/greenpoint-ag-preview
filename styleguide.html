/ Style guide

A system,
not a screen.

The tokens, elements, and patterns that compose the GreenPoint Ag landing page. Use this as the source of reference when building new pages. All tokens are defined as CSS custom properties on :root in styles.css — never hard-code a hex or pixel value.

Color

Tinted neutrals + brand greens. --c-green is the primary action color; --c-green-light is reserved for the lighter highlight forms in the wordmark and never for body UI. Always use the token, never the hex.

Usage

  • Surfaces: --c-white for primary content, --c-cream-lt for subtle inset blocks, --c-cream / --c-beige for warmer support surfaces.
  • Text: --c-text for body, --c-black for display headings, --c-text-muted for secondary copy.
  • Action: --c-green primary, --c-green-dark hover/pressed. Never combine green-light with green-dark on the same element except inside the wordmark.
  • Reverse: on green or black surfaces, text is --c-white and focus ring becomes --c-cream.

Typography

One typeface, three weights. Open Sans 300 / 400 / 600 / 700. Display sizes use weight 300 with negative letter-spacing; body uses 400. Italic emphasis on green words is the brand's signature voice cue.

Voice patterns

  • Italic emphasis in green: wrap one short clause inside a display heading in <em> with color: var(--c-green-dark). Sparing — once per heading, never two clauses.
  • Body line-length: 32–65ch. Use max-width: 32ch on card bodies, 65ch on long-form paragraphs.
  • No em dashes in user-facing copy. Use commas, colons, or parentheses.

Section padding

Sections use a responsive vertical pad: clamp(3rem, 6vw, 6rem). Horizontally, always pad with var(--gutter). Connect / hero / footer use heavier vertical pads (var(--space-7) minimum).

Layout

Single source: --container caps content at 1320px, --gutter sets responsive side padding.

Breakpoints

  • 30em · 480px — small phone → large phone
  • 48em · 768px — phone → tablet (most layouts pivot here)
  • 64em · 1024px — tablet → desktop (root font bumps to 18px)

Cards and large surfaces use --radius-md. Pills (utility nav, zip search, buttons) use --radius-pill. Code samples and small inset blocks use --radius-sm.

Shadows

--shadow-sm
--shadow-md
--shadow-lift

Shadows are functional, not decorative. --shadow-sm = subtle separation (swatches, code blocks). --shadow-md = standard card resting state. --shadow-lift = hover/active lift.

Motion

Three ease curves cover everything. Never use bounce or elastic. Durations: 150–350ms for state changes, 500–800ms for entrance reveals. Exit animations run at ~75% of entrance duration.

--ease-out
--ease-out-quart
--ease-out-expo

Rules

  • Use --ease-out for color / opacity transitions.
  • Use --ease-out-quart for transforms and most state changes (cards, links).
  • Use --ease-out-expo only for confident, decisive moments — accent rules expanding, hero reveals.
  • Always respect prefers-reduced-motion; motion is enhancement, never load-bearing.

Buttons

Three variants. All buttons inherit the letter-stagger label animation on hover (handled automatically by script.js; just use the .btn class). Buttons include their arrow icon when leading to a destination, omit when triggering an action.

<a class="btn btn--green">
  Customer Login
  <svg ...>arrow</svg>
</a>

<a class="btn btn--outline-dark">Who We Are</a>
Create Your Account 
<!-- Use on dark/green surfaces only -->
<a class="btn btn--outline-light">Create Your Account</a>

Rules

  • --green: the primary action. Use sparingly — never more than one per visible region.
  • --outline-dark: secondary on light surfaces. Hover inverts to solid black/white.
  • --outline-light: secondary on dark/green surfaces. Hover inverts to white/black.
  • Arrow icon: present when the action is navigation. On hover, arrow slides 3px right.

Reveal on scroll

Any element added to the targets NodeList in script.js animates in (fade + 20px lift) when it enters the viewport. Progressive enhancement: elements are visible by default; the script tags them .is-pending at observe time and switches to .is-visible on intersection. Disabled when prefers-reduced-motion is set.

// To opt new elements into the reveal:
// add their selector to the querySelectorAll list in script.js
